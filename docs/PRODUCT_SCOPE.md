## 1. 产品一句话定义

Arena 是一个“可验证人群共识与调研网络”。

平台由两个并行运行但信息隔离的层组成：

- `裁决层`：负责分发命题或问卷、收集回答、做回答质检、形成有效样本，并计算群体结果。
- `验证层`：负责围绕未来将公开的群体结果进行下注或验证，并在触发条件满足后统一结算。

## 2. 产品主张

- Arena 不是单纯的问卷平台，也不是单纯的竞猜或预测平台。
- 它的核心能力是“隐藏式共识生产 + 围绕结果的验证/交易”。
- 同一用户可以对同一命题同时回答和下注。
- 系统隔离的不是用户身份，而是实时信息流。
- 开奖前只公开进度，不公开任何方向性信息。

## 3. 目标用户

### 3.1 需求方

- 品牌方
- 公司
- 研究机构
- 社区 / DAO
- 内容平台

他们需要的核心输出包括：

- 可验证样本
- 高质量问卷答案
- 群体共识结果
- 持续更新的偏好趋势
- 可验证、可结算的结果输出

### 3.2 参与者

- `Respondent`：接收命题或问卷，提交回答，获得奖励，沉淀标签和信誉。
- `Trader / Verifier`：围绕未来将公开的群体结果下注，等待开奖结算。
- `Requester`：发布任务，定义样本条件，设置预算，查看结果和报告。

## 4. 产品不变量

- 裁决层不能看到市场方向、下注分布、价格倾向或盘口走势。
- 验证层不能看到实时回答倾向、当前领先方、实时比例或内部趋势。
- 开奖前允许公开的信息只包括时间进度、有效样本进度、是否达到开奖门槛，以及上一周期结果（仅滚动型命题）。
- 平台必须保留派单权，不能把核心采样逻辑建立在用户自由抢题上。
- 用户回答生成的标签只能作用于未来任务，不能即时反向改变当前命题样本。
- 奖励必须按“有效贡献”结算，不能按提交次数直接发放。
- 改票是更新状态，不是重复完成任务，不能重复领取完成奖励。

## 5. 产品范围

### 5.1 整体产品范围

- 统一顶层对象为 `Task / Proposition`
- 支持三种任务模式：`Consensus`、`Survey`、`Hybrid`
- 支持两类命题形态：`Non-rolling`、`Rolling`
- 系统需要覆盖的核心模块：
- 用户与身份模块
- 命题模块
- 分发模块
- 回答模块
- 质检模块
- 标签模块
- 市场模块
- 开奖与结算模块
- 奖励模块
- 管理与运营模块

### 5.2 MVP 范围

- 只做二选一共识命题
- 只做单题短问卷
- 只做非滚动型命题
- 验证层只做简单下注和一次性结算
- 平台负责派单、回答提交、基础质检、有效样本计数和统一开奖
- 奖励链路先做“待结算 -> 审核后发放”
- 标签先做基础兴趣标签和质量 / 信誉标签
- 前端在开奖前只展示进度，不展示方向

### 5.3 MVP 后续扩展

- 滚动型命题与周期结算
- 多题问卷
- 混合任务
- 样本不足时保留上一周期结果的展示逻辑
- 报告导出
- 需求方后台
- 更复杂的风险控制和画像系统

## 6. 核心用户流程

### 6.1 平台主链路

1. 平台或任务方创建命题，配置标题、描述、选项、样本条件、时间参数、奖励预算，以及是否开启验证层。
2. 命题上线后，同时进入裁决层和验证层。
3. 裁决层向目标用户派发任务并持续收集回答。
4. 回答进入质检，只有 `valid` 和 `partial_valid` 计入有效样本。
5. 验证层持续接收下注，但开奖前只能展示进度信息。
6. 非滚动型命题在“达到最短时间且达到最低有效样本”或“达到最大截止时间”后触发开奖。
7. 系统读取裁决层聚合结果，在验证层完成结算，并发放回答奖励、更新标签与信誉。
8. 非滚动型命题结束；滚动型命题在后续阶段进入下一周期。

### 6.2 Respondent 流程

1. 收到平台派发的命题或问卷
2. 提交回答
3. 等待质检
4. 有效回答计入样本
5. 获得奖励并更新信誉或标签

### 6.3 Trader / Verifier 流程

1. 进入某个命题市场
2. 查看可下注选项和进度信息
3. 在看不到方向信息的前提下下注
4. 等待开奖
5. 开奖后完成结算

## 7. MVP 明确不做

- 复杂长问卷
- 开放题深度分析
- 第一阶段内的滚动型命题
- 复杂 AMM
- 复杂订单簿
- 多资产下注
- 重型模型风控
- 复杂会员体系
- 复杂奖励体系
- 开奖前暴露任何方向性中间态

## 8. MVP 成功标准

- 可以完整跑通一个非滚动型二选一命题的创建、派发、回答、质检、聚合、开奖和结算链路。
- 同一用户可以同时回答和下注，但不会在开奖前拿到任何方向性反馈。
- 平台可以稳定识别有效样本，并基于有效样本生成最终结果。
- 验证层可以围绕最终结果完成一次性结算。
- 回答奖励可以在审核后发放，标签和信誉可以进入下一轮任务分发。
